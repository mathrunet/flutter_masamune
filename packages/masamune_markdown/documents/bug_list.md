# Android実機での不具合

[x] バックスペースボタンで文字が削除されない
[x] カーソルがありソフトウェアキーボードが表示されている段階で文字をタップするとカーソルが解除される。再度タップするとカーソルが当たる。→本来は他の文字をタップするとそこにカーソルが当たるようになる（カーソルを解除しない）
[x] カーソルが当たっていない段階で文字を長押しすると反応しない
[x] ソフトウェアキーボードが表示されている場合、ツールバーの上の部分で文字が表示されるべき場所に文字が表示されない（トリムされているように見える）選択されている場合は表示される。
[x] スクロールしようとするとスクロールできず、文字が選択と非選択を高速で繰り返す挙動をする
[x] 各ブロックの間をタップしてしまうと一番下の文字にカーソルが合ってしまう。ブロックの間をタップしてもそこに一番近いブロックの文字にカーソルが合うようにしてほしい。→この影響で長押しでの選択後、そのまま指を移動しようとすると少ししか動かしていないのに一番最後の文字まで選択されてしまう。
[x] ３行以上文字がある場合、途中の空行にカーソルを合わせて文字を入力した後改行ボタンをタップすると途中の行をすべて飛ばして最後の行の最後にカーソルが移動してしまう。想定では現在の行の下に空行を挿入してそこにカーソルを合わせるようになるはず。
[x] 見出しなどcanIndent=falseのBlockValueはインデントできないようにしているはずだが、それらのブロックにカーソルがあるときにインデントボタンがアクティブになっているので非アクティブにしてほしい
[x] Androidの場合、OSの戻るボタンでキーボードを閉じれるが一旦閉じてしまうと文字をタップしてもキーボードが表示されなくなる
[x] 番号リストの不具合
  - 連番にならない
  - 番号が下にずれている
  画像の例だと
  ```
  1. あかさ
  2. たなは
    a. はまや
    b. らわ
  3. さはら
  ```
  になるはず
[x] チェックボックスと文字の間にスペースが欲しい
[x] チェックボックスをタップしてもチェックが入らない
[x] ２個目以降のチェックボックスをタップすると1個目のチェックボックスにチェックが入る
[x] チェックボックスにチェックが入ったときは文字に打ち消し線を追加するようにして
[x] IME入力をして確定する前にインデントなどのボタン操作をしたりとかすると入力の挙動がおかしくなる。IME入力途中の場合、IMEによる操作以外のフィールドへのすべての操作（ツールボタンタップやカーソルの移動、選択など）が行われた場合、強制的にIMEの文字入力を確定して
[x] IMEを確定せずそのまま削除した場合最後の文字を削除するタイミングでカーソルがずれる。例えば「あかさ」と入力されているところに追加で「はまや」と入力して「あかさはまや」を入力した後、IMEを確定せず「や」から順に削除していくと「あかさはま」→「あかさは」となるまではOKだが、この後にカーソルが「さ」の右側にカーソルが移動してしまい、そのまま削除すると「さ」が削除され「あかは」となってしまう。「は」が削除され「あかさ」となるべき。
[x] 新しく追加で「１行以上の空行入力」→「文字を入力」→「入力した文字を削除し行を削除」→をしていく場合、入力した文字が削除された後１行空行を削除して文字が入力されていく挙動になるはずが、「カーソルが消える」→「文字入力」→「１行飛ばした行に文字が入力される」という挙動になってしまう。
例（現状）
```
あかさ


はまや|
```
↓
```
あかさ


|
```
↓
```
あかさ
|

```
↓
```
あかさ
らわ|

```
[x] カーソルがあたっていない（ソフトウェアキーボードが表示されていない状況）で長押しでテキストを選択したときにソフトウェアキーボードが表示されない。想定としてはキーボードが表示される。
[] 長押しで選択した後範囲外の場所をタップしてフォーカスを解除した場合でも長押し状態が続く。この場合はフォーカスが解除されたと同時に長押し状態も解除してほしい。
[] Androidキーボードのカーソルボタン（左右）をタップしてもカーソルが動かない
[] 若干各ブロックのbottomの余白が大きい
[] コードの背景をborderRadiusで丸くして。プロパティは設定した。
[] Bouncingのスクロールをして限界まで上下を動かすと上下で文字がクリップされてしまう。またそれと合わせてFormがexpanded担っているにも関わらず、expandedされたエリアをタップしても文字入力が可能にならない。おそらくスクロール部分でMarkdownFieldエリアが終わってしまっていてタップ対象になっていないのが原因
[] 引用もコードと同じようにマルチラインで改行しても別ブロックを作成しないようにする
