# Android実機での不具合

[x] バックスペースボタンで文字が削除されない
[x] カーソルがありソフトウェアキーボードが表示されている段階で文字をタップするとカーソルが解除される。再度タップするとカーソルが当たる。→本来は他の文字をタップするとそこにカーソルが当たるようになる（カーソルを解除しない）
[x] カーソルが当たっていない段階で文字を長押しすると反応しない
[] ソフトウェアキーボードが表示されている場合、ツールバーの上の部分で文字が表示されるべき場所に文字が表示されない（トリムされているように見える）選択されている場合は表示される。
[x] スクロールしようとするとスクロールできず、文字が選択と非選択を高速で繰り返す挙動をする
[] 各ブロックの間をタップしてしまうと一番下の文字にカーソルが合ってしまう。ブロックの間をタップしてもそこに一番近いブロックの文字にカーソルが合うようにしてほしい。→この影響で長押しでの選択後、そのまま指を移動しようとすると少ししか動かしていないのに一番最後の文字まで選択されてしまう。
[] ３行以上文字がある場合、１行目にカーソルを合わせて見出しやリストのブロックを新規追加して文字を入力した後改行を行おうとすると１行飛ばして２行先の文字の一番最後にカーソルが移る本来であれば、見出し１ブロックの下に新しい空のテキストブロックが作成されそこにカーソルが合う想定
[] 見出しなどcanIndent=falseのBlockValueはインデントできないようにしているはずだが、それらのブロックにカーソルがあるときにインデントボタンがアクティブになっているので非アクティブにしてほしい
[] Androidの場合、OSの戻るボタンでキーボードを閉じれるが一旦閉じてしまうと文字をタップしてもキーボードが表示されなくなる
[] 番号リストの不具合
  - 連番にならない
  - 番号が下にずれている
  画像の例だと
  ```
  1. あかさ
  2. たなは
    a. はまや
    b. らわ
  3. さはら
  ```
  になるはず
[] コードのブロックの背景色を色が変わっていないが描画されていないだけか、設定が抜けているだけか
[] 引用の左側にインデントが適用されない。また、左側にボーダーが表示されるはずだがそれがない。描画されていないだけか、設定が抜けているだけか
[] チェックボックスと文字の間にスペースが欲しい
[] チェックボックスをタップしてもチェックが入らない
[] Androidキーボードのカーソルボタン（左右）をタップしてもカーソルが動かない
